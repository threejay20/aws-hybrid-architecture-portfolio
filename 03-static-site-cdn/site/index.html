How to read these notes

Each project is documented with the same structure so it’s easy to review for interviews and exam prep.

Rule of thumb: If you can explain the trade-offs (cost, security, failure handling), you’re not memorizing AWS — you’re thinking like a Solutions Architect.
Exam-style mental triggers
Need fanout?
SNS (one event → multiple subscribers)
Need buffering?
SQS (spikes, decoupling, durability)
Need $0 idle?
Lambda + managed services
Need failure isolation?
DLQ (poison-pill protection)
Need global latency reduction?
CloudFront (edge caching)
